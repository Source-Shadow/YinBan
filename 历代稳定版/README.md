# 历代稳定版

本目录存放 **AI 影伴 (YinBan)** 项目的历代稳定版 APK 及对应更新说明。

---

## 命名规范

每个版本需要放置 **两个文件**，命名格式如下：

| 文件类型 | 命名格式 | 示例 |
|---------|---------|------|
| APK 安装包 | `YinBan_vX.Y.apk` | `YinBan_v1.0.apk` |
| 更新说明 | `vX.Y更新.md` | `v1.0更新.md` |

> **版本号规则**：`vX.Y` — `X` 为主版本号（重大功能变更），`Y` 为次版本号（Bug 修复 / 小改进）。
>
> **命名大小写**：APK 文件名统一使用 `YinBan`（与代码包名 `com.yinban.ai` 一致）。已有的 `YingBan` 拼写也可接受，新版本请统一为 `YinBan`。

---

## 如何添加新版本

当你打出一个稳定的 APK 后：

1. **复制 APK 到本目录**，按上述格式重命名
2. **新建更新说明** `vX.Y更新.md`，参考下面的模板
3. **更新本文件顶部"版本列表"表格**，新增一行
4. 提交到 Git：
   ```bash
   git add 历代稳定版/
   git commit -m "release: vX.Y 稳定版"
   git push origin main
   ```

### 更新说明模板

```markdown
# AI 影伴 vX.Y 更新通知

> **发布日期**：YYYY-MM-DD
> **编译状态**：✅ BUILD SUCCESSFUL
> **APK 文件**：`历代稳定版/YinBan_vX.Y.apk`

---

## 🎨 外观变化
- 用户能看到、能感知到的界面改动

## 🐛 问题修复
- 修复了哪些 UI 或交互问题

## 🆕 功能变化（如有）
- 简要说明新增功能

---

## 🚀 安装
```bash
adb install 历代稳定版\YinBan_vX.Y.apk
```
```

## 注意事项

- **每个稳定版必须同时有 APK + 更新说明**，缺一不可
- APK 必须是 **确认可正常运行** 的版本，不要放中间测试版
- 更新说明重在描述**用户能看到、能感知到的变化**（界面改动、交互体验、文案调整），不需要罗列技术实现细节
- 旧版本 **不要删除**，保留完整历史追溯
