## 🆕 首次发布

AI 影伴 v1.0 — 端到端核心链路走通，患者端 & 监护人端双端可用。

### 核心功能

| 功能 | 说明 |
|------|------|
| 🤝 **房间号配对** | 患者和监护人输入同一房间号即可互通 |
| 📹 **实时视频** | ESP32-CAM → MJPEG HTTP → 监护人端 WebView 显示 |
| 💬 **消息收发** | WebSocket 双向文字消息（断线自动重连 + 心跳保活） |
| 📍 **位置共享** | GPS 实时上报 + 隐私模式模糊化（~1.1km 精度） |
| 🆘 **一键 SOS** | 长按 2 秒触发，自动附带位置 + 视频流地址 |
| 🎛️ **设备控制** | 摄像头/麦克风/耳机独立开关，患者最终控制权 |
| 🎤 **TTS 语音** | AI 指令文字转语音播报 |

### 技术架构

```
Kotlin (Android)  ←→  Python FastAPI (WebSocket)  ←→  ESP32-CAM (MJPEG)
     患者端 + 监护人端             消息中转 + 房间管理              视频推流
```

| 类别 | 选型 |
|------|------|
| 语言 | Kotlin |
| 最低 SDK | 26 (Android 8.0) |
| UI | Material Design 3 + ViewBinding |
| 网络 | OkHttp 4.12 WebSocket |
| 序列化 | Gson 2.10 |
| TTS | Android TextToSpeech |

### 页面结构

```
MainActivity (Splash)
  ├── LoginActivity (角色选择 → 账号密码 → 配对码)
  ├── PatientActivity (患者端核心)
  │     ├── 设备状态面板
  │     ├── SOS 紧急按钮
  │     └── AI 指令显示
  └── GuardianActivity (监护人端)
        ├── MJPEG 实时画面 (WebView)
        ├── 位置地图
        └── 预警弹窗
```

### 支持的 9 种消息类型

`ping/pong` · `room_status` · `stream_start` · `stream_url` · `location_update` · `device_status` · `manual_message` · `device_control_request` · `ai_voice_command`

## 🚀 安装

```bash
adb install 历代稳定版\YingBan_v1.0.apk
```

### 使用前提

1. 电脑启动后端服务器（`后端/main.py`）
2. 手机与电脑连同一 WiFi 或电脑热点
3. 防火墙放行 8000 端口

---

## ⚠️ 已知限制

- 不支持用户注册/登录系统（使用配对码直连）
- 无数据库持久化（房间在内存中，重启清空）
- 仅单摄像头画面，不支持多路视频
- 视频流为 MJPEG（延迟约 2-5s），非 WebRTC 实时通话
