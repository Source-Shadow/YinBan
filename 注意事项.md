## 注意事项

本仓库是 AI影伴 项目的共享参考资料库。每个人把自己负责部分的最新进度、文档、代码片段等放在自己的文件夹里，供其他队员及其 AI 参考。**各项目之间不会在这个仓库里互联互通**，纯作资料存档和参考之用。

### 文件夹结构

每个人在根目录下建一个自己的文件夹，里面随便放，按自己习惯组织即可：

```

├── 后端/
│   ├── 进度.md
│   ├── 接口设计.md

│   ├── UI设计稿/
│   └── 前端进度.md
├── 硬件/
│   └── ...

```

> 每人只碰自己的文件夹，不要改别人的东西。

### 日常操作（重要！每个人都必须这样做）

#### 1. 开始工作前 — 拉取最新内容

```bash
git pull origin main
```

**不 pull 直接 push 会被拒绝，必须先拉取。**

#### 2. 更新自己的内容

在自己的文件夹里写 md、放文件，随便你怎么组织。

#### 3. 提交并推送

```bash
git add .
git commit -m "张三: 更新后端进度"
git push origin main
```

提交信息写清楚**谁 + 干了什么**就行。

> **第一次 push 时**可能会提示 `no upstream branch`，按提示执行：
> ```bash
> git push --set-upstream origin main
> ```
> 之后再用 `git push` 就不需要了。

---

### 冲突怎么处理

当你 `git pull` 时如果提示冲突（conflict），说明你改的文件别人也改了。

**避免冲突的方法：**

- 每个人只动自己文件夹里的东西
- 勤 pull，及时 push
- 不要编辑别人的文件

如果还是冲突了，打开冲突文件，会看到类似这种标记：

```
<<<<<<< HEAD
你的内容
=======
别人的内容
>>>>>>> ...
```

手动保留想要的内容，删掉 `<<<<<<<`、`=======`、`>>>>>>>` 这些标记，然后：

```bash
git add .
git commit -m "解决冲突"
git push origin main
```

### 核心规则

1. **每人只在自己的文件夹里操作**，别碰别人的内容
2. **先 pull 再 push**，养成习惯
3. **提交信息写清楚**，让别人一眼知道这次更新了什么
4. **及时更新**，尤其是开会后、有重要进展时
