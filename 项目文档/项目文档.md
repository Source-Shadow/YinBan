## 一、项目介绍

为轻中度成年自闭症患者打造的可穿戴AI辅助工具，部分替代"影子老师"功能。

**目标用户**：具有一定语言理解能力、可进行基础社会活动、但在社交应答和情境判断上存在困难的轻中度成年自闭症患者。

**使用方式**：患者佩戴摄像头（胸前）+ 骨传导耳机外出，手机App实时上传音视频到服务器，AI分析后：

- 正常情况 → 耳机给患者语音提示（辅助社交）
- 检测到异常 → 推送预警到监护人手机

**两大核心功能**：

| 功能  | 说明                                |
| --- | --------------------------------- |
| 预警  | AI检测到异常行为/情绪/环境风险 → 监护人端App弹窗+通知栏 |
| 陪伴  | 社交场景需要辅助 → 患者端耳机播放AI语音 + 屏幕显示文字   |

## 二、项目框架

```
患者(摄像头+骨传导耳机)
        │
        ▼
  患者手机App（Flutter，患者模式）
        │
        │  WebSocket + 视频推流
        ▼
  服务器（Python FastAPI + SRS流媒体）
  运行在Windows笔记本上
        │
        ├──────────────┐
        ▼              ▼
   AI分析Pipeline    监护人手机App
   (ASR+视觉+LLM+TTS) (Flutter，监护人模式)
        │              - 看实时画面
        ▼              - 收预警通知
   结果回传            - 看地图定位
   - 预警→监护人      - 发消息给患者
   - 提示→患者
```

**数据传输链路**：

- 摄像头 → 手机：WiFi（RTMP推流）
- 手机 → 服务器：WiFi（同局域网）/ 4G、5G（Cloudflare Tunnel公网）
- 手机 → 耳机：蓝牙（A2DP/HFP）
- 服务器 → 监护人手机：WiFi（同局域网）/ 4G、5G

**硬件选型**：

| 设备  | 型号                           | 价格       |
| --- | ---------------------------- | -------- |
| 摄像头 | ESP32-CAM（WiFi推流，640×480）    | ¥30-60   |
| 耳机  | 韶音 OpenMove 骨传导（不入耳，能同时听环境音） | ¥300-600 |

**连接方式**：患者端输入房间号（如 DEMO001）→ 连服务器 ← 监护人端输入同一房间号 → 自动配对。

### 三套接口

**接口A：App ↔ 服务器（WebSocket JSON）**

连接地址：`ws://<IP>:8000/ws?room=<ROOM_ID>&role=<patient|guardian>`

消息格式：

```json
{"msg_id": "uuid", "type": "类型", "data": {}, "timestamp": "ISO8601"}
```

消息类型：

| type                   | 方向         | 说明               |
| ---------------------- | ---------- | ---------------- |
| join_room              | App→Server | 加入房间             |
| room_status            | Server→App | 房间状态变化           |
| location_update        | 患者→服务器→监护人 | GPS位置            |
| frame_upload           | 患者→服务器     | 视频帧(JPEG base64) |
| audio_chunk            | 患者→服务器     | 音频片段(base64)     |
| stream_start           | 患者→服务器     | 开始推流             |
| stream_url             | 服务器→监护人    | 推流地址             |
| device_status          | 患者→服务器→监护人 | 设备开关状态           |
| ai_prompt              | 服务器→患者     | AI提示(文字+语音)      |
| alert                  | 服务器→监护人    | 预警通知             |
| manual_message         | 监护人→患者     | 手动发消息            |
| device_control_request | 监护人→患者     | 请求开关设备           |
| ping / pong            | 双向         | 心跳               |

**接口B：服务器 ↔ AI Pipeline（Python函数调用）**

C的代码 `from ai import` 调用D写的三个函数：

```python
# 视频帧分析（每2秒一次）
def analyze_frame(image_bytes: bytes, patient_id: str, context: dict) -> AnalysisResult

# 音频分析（每5秒一次）
def analyze_audio(audio_bytes: bytes, patient_id: str, context: dict) -> AnalysisResult

# 综合风险评估
def assess_risk(patient_id: str, recent_frames: list, recent_asr_text: list,
                location: dict, device_status: dict) -> AnalysisResult

# 返回结构
class AnalysisResult:
    alert: Optional[Alert]    # 有则推给监护人
    prompt: Optional[Prompt]  # 有则推给患者

class Alert:
    type: str        # emotion | conflict | stagnation | environment | geofence | device
    severity: str    # red | yellow | green
    message: str

class Prompt:
    text: str
    audio_base64: Optional[str]  # TTS合成的语音
```

## 五、项目特点

**1. 不做数据库**

基础版不用任何数据库。通过房间号直接配对，像进Zoom会议室。不需要用户注册、登录、绑定、历史记录。数据全部实时流转，不落盘存储。

**2. Windows笔记本当服务器**

不租云服务器。用一台Windows笔记本运行 Python FastAPI + SRS 流媒体服务。不需要学 Linux、SSH、Docker、Nginx、安全组配置。调试时直接在本机看日志，出问题立刻改。

**3. 设备、手机、电脑同WiFi**

同一局域网下，手机直接连电脑IP（如 `192.168.1.100:8000`）通信。视频推流走局域网，延迟低、不消耗手机流量。需要外出演示时，用 Cloudflare Tunnel（免费）获得公网域名，手机用4G也能连回电脑。

**4. 设备独立开关，患者最终控制**

摄像头和耳机在App中可分别独立开关。四种组合：全开、关摄像头（需隐私）、关耳机（需安静）、全关（休息）。患者拥有最终控制权，监护人可远程请求开启但需患者确认。

**5. 数据不经过硬件直连4G**

所有数据走"硬件 → 患者手机 → 服务器"路径，不跳过手机。手机做中转的好处：手机每人都有不需要额外硬件、App控制采集和上传、统一的数据出口方便管理。

**6. 接口先行，并行开发**

App ↔ 服务器（WebSocket JSON）、服务器 ↔ AI（Python函数调用）、Flutter内部（共享模块）三套接口在开发前冻结。五个人按接口独立开发，各自写自己的部分，最后拼在一起。改接口必须全员通知。
