# AI影伴 — 前端设计文档

> Android 原生 Kotlin + Material Design 3 (ViewBinding)，双端合一 App
> 目标用户：轻中度成年自闭症患者 + 其监护人

---

## 一、整体架构概览

### 1.1 完整数据流

```
╔══════════════════════ 上行：数据采集 ══════════════════════╗
║                                                          ║
║  [ESP32-CAM]────MJPEG推流(WiFi)────┐                     ║
║         (胸前摄像头)               │                      ║
║                                   ▼                      ║
║  [骨传导耳机]───蓝牙──→[患者手机]──WebSocket──→  ┌─────────┐ ║
║   (麦克风采音)          (Kotlin)               │ 服务器   │ ║
║                            │                   │(FastAPI │ ║
║                            │ GPS/设备状态/SOS   │ + SRS)  │ ║
║                            └──WebSocket──────→  │         │ ║
║                                                 └────┬────┘ ║
╚══════════════════════════════════════════════════════╪══════╝
                                                       │
                                                    AI Pipeline
                                                 (ASR语音→视觉→LLM→TTS)
                                                       │
╔══════════════════════ 下行：AI结果下发 ═══════════════╪══════╗
║                                                       │      ║
║       ┌───────────────────────────────────────────────┘      ║
║       │                                                      ║
║       ▼                                                      ║
║  ┌─────────┐                                                 ║
║  │ 服务器   │                                                 ║
║  └────┬────┘                                                 ║
║       │                                                      ║
║       ├──WebSocket──→[监护人手机]  预警弹窗 + 定位 + 消息     ║
║       │              (Kotlin)                                ║
║       │                                                      ║
║       ├──MJPEG HTTP──→[监护人手机]  实时视频画面（WebView）    ║
║       │                                                   ║
║       └──WebSocket──→[患者手机]──TTS合成──蓝牙──→[骨传导耳机] ║
║                       AI语音指令           (患者听到提示)     ║
╚══════════════════════════════════════════════════════════════╝
```

**一句话总结**：所有数据汇总到服务器 → AI 分析 → 预警推给监护人、提示推给患者。视频走 MJPEG（ESP32-CAM 直连 Flask），音频和信令走 WebSocket（手机中转）。

## 四、视频流传输链路（📍 已联调测试通过）

### 4.1 数据流向

```
ESP32-CAM                  Flask MJPEG              监护人手机
(胸前摄像头)                服务器(硬件同学)          (GuardianActivity)
    │                          │                        │
    │── WiFi 推流 ────────────→│                        │
    │   MJPEG HTTP             │                        │
    │   192.168.137.83:5000    │                        │
    │                          │←─ stream_start ───────│ 患者触发
    │                          │── stream_url ────────→│ 服务器下发URL
    │                          │                        │
    │                          │←─ HTTP GET ───────────│ WebView.loadUrl()
    │                          │── MJPEG frames ──────→│ 实时画面显示
```

### 4.2 实现方式（v1.0 刚刚简化）

```kotlin
// GuardianActivity.kt — 之前用 loadDataWithBaseURL + 内联 HTML，现已简化为：
private fun loadMJPEG(url: String) {
    binding.webVideo.apply {
        visibility = View.VISIBLE
        stopLoading()
        loadUrl(url)  // 直接加载 MJPEG URL
    }
    binding.tvGuardianVideoPlaceholder.visibility = View.GONE
}
```

| 对比 | 旧方案 | 新方案 |
|------|--------|--------|
| 加载方式 | `loadDataWithBaseURL` + 内联 HTML | `loadUrl(url)` 直接加载 |
| HTML 代码 | 内嵌 `<img>` 标签页 | 不需要，浏览器原生渲染 MJPEG |
| 代码行数 | ~20 行 | 4 行 |
| 性能 | WebView 先解析 HTML 再请求图片 | 浏览器直接解码 MJPEG 流 |

### 4.3 默认推流地址

```
http://192.168.137.83:5000/video_feed
```

这是硬件同学（ESP32-CAM + Flask）的 IP，在 `后端/main.py` 第 22 行硬编码。可通过环境变量覆盖：
```powershell
$env:MJPEG_STREAM_URL = "http://<新IP>:5000/video_feed"
python main.py
```

### 4.4 视频流消息协议

| 消息 | 方向 | 说明 |
|------|------|------|
| `stream_start` | 患者 → 服务器 | `{"stream_type":"video"}` 患者开关推流 |
| `stream_url` | 服务器 → 监护人 | `{"url":"http://192.168.137.83:5000/video_feed","stream_type":"video","status":"live"}` |
| `sos_alert` | 患者 → 监护人 | 服务器自动补 `stream_url` 后再转发 |

## 五、消息协议与接口

### 5.1 连接信息

```
ws://<服务器IP>:8000/ws?room=<房间号>&role=<patient|guardian>
```

所有消息通用外壳：`{"msg_id":"uuid","type":"类型","data":{...},"timestamp":"ISO8601"}`

### 5.2 全部 13 种消息类型

| 类别 | type | 方向 | 处理方式 |
|------|------|------|----------|
| 心跳 | `ping` / `pong` | 双向 | 服务器回复 pong |
| 房间 | `room_status` | Server→App | 服务器生成：`waiting` / `paired` / `disconnected` |
| 推流 | `stream_start` | 患者→Server | 服务器生成 `stream_url` 发给监护人 |
| 推流 | `stream_url` | Server→监护人 | 服务器填 MJPEG 地址 |
| 转发⚡ | `location_update` | 患者→监护人 | 带 `lat/lng/accuracy/is_privacy_mode/is_sos` |
| 转发⚡ | `device_status` | 患者→监护人 | 带 `camera/headphone/microphone/battery/network_type` |
| 转发⚡ | `manual_message` | 双向 | `content + from_role` |
| 转发⚡ | `device_control_request` | 双向 | `device + action(on/off)`，需患者确认 |
| 转发⚡ | `call_request` / `call_response` | 双向 | WebRTC 信令 |
| SOS | `sos_alert` | 患者→监护人 | 服务器补 `stream_url` 后转发 |
| AI⚡ | `ai_voice_command` | Server→患者 | `voice_text + action_code + priority`，TTS 朗读 |
| AI⚡ | `danger_detected` | Server→患者 | `danger_type + confidence`，>0.7 自动 SOS |

> 详细字段定义见 `后端/接口对接单.md`（前后端唯一标准）。

### 5.3 前后端交互序列（关键场景）

```
# 日常配对
患者 App ──connect(room=X, role=patient)──→ 服务器 ← room_status(waiting)
监护人 App ──connect(room=X, role=guardian)──→ 服务器 → room_status(paired) 双方

# 患者开视频 → 监护人看到实时画面
患者开关 🎥 打开
  → stream_start{video} → 服务器
  → stream_url{url} → 监护人
  → 监护人 WebView.loadUrl(stream_url) → 实时 MJPEG 画面显示

# 患者 SOS → 监护人紧急响应
患者长按 SOS 按钮 → 确认
  → sos_alert{lat,lng,message} + location_update(isSos=true) → 服务器
  → 服务器自动补 stream_url
  → 监护人端：红色弹窗 + 精确位置 + 自动加载实时画面
```

## 八、动态演示

### 8.1 动画规范

| 场景 | 实现 | 时长 |
|------|------|------|
| 页面过渡 | `overridePendingTransition(R.anim.fade_in, R.anim.fade_out)` | 300ms |
| AI 推送卡片 | `translationY(-200→0) + alpha(0→1)` | 300ms 滑入，3s 后自动收起 |
| SOS 脉冲 | `ValueAnimator.ofFloat(1f → 0.4f)` 重复 ×4 | 500ms/次，2s 自动停止 |

### 8.2 动画禁忌（针对自闭症患者）

| ❌ 不使用 | ✅ 替代 |
|-----------|--------|
| `OvershootInterpolator`（弹性回弹） | `DecelerateInterpolator` |
| `AnticipateInterpolator`（先反向再正） | `AccelerateInterpolator` |
| 快速闪烁（<200ms） | 脉冲间隔 ≥500ms |
| 自动轮播/Banner | 静态卡片 |
| ViewPager 左右滑动 | 点击切换 + fade 过渡 |

### 9.4 关键组件速查

### 9.5 无障碍清单

| 检查项 | 要求 |
|--------|------|
| contentDescription | 所有 ImageView、ImageButton 必须设置 |
| 触摸目标 | 全部 ≥ 48dp |
| 文字单位 | 全部 `sp` 跟随系统字体缩放 |
| 色彩对比度 | 正文 ≥ 4.5:1，大文字 ≥ 3:1 |
| TalkBack | 连接断开、SOS 触发等关键事件主动播报 |
| 动画 | 无弹性/弹簧/闪烁动效 |
| 图标+文字 | 导航按钮、操作按钮同时显示图标和文字 |
